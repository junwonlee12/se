#  무인 카페 키오스크 시스템 요구사항 분석서

본 문서는 인건비 절감 및 매장 운영 효율화를 위한 **스마트 무인 카페 키오스크(Kiosk) 시스템**의 요구사항 정의서입니다.

---

## 1. 개요
* **프로젝트명:** 스마트 무인 카페 키오스크(Kiosk) 시스템 구축
* **목적:** 고객이 직접 메뉴 주문, 결제, 스탬프 적립을 처리하는 터치스크린 기반 시스템 개발
* **대상 사용자:** 매장 방문 고객, 매장 관리자

## 2. 시스템 구조 및 흐름 (Workflow)
[대기 화면] ➔ [메뉴 선택 (옵션 추가)] ➔ [장바구니 확인] ➔ [스탬프 적립/사용] ➔ [결제 (카드/페이)] ➔ [주문서/영수증 출력] ➔ [대기 번호 호출]

## 3. 기능적 요구사항 (Functional Requirements)


| 요구사항 ID | 기능 분류 | 요구사항 상세 내용 | 중요도 |
| :--- | :--- | :--- | :--- |
| **FR-01** | **메뉴 탐색** | • 음료(Hot/Ice), 디저트, MD 상품 카테고리를 탭 형태로 제공한다.<br>• 품절된 메뉴는 선택할 수 없도록 '품절' 표시 및 비활성화한다. | 상 |
| **FR-02** | **옵션 선택** | • 메뉴 선택 시 팝업창을 통해 샷 추가, 시럽 추가, 얼음 양(적게/보통/많이)을 선택할 수 있어야 한다. | 상 |
| **FR-03** | **주문 담기** | • 선택한 메뉴와 옵션, 수량을 장바구니에 담고 실시간으로 총 금액을 화면 하단에 표시한다. | 최상 |
| **FR-04** | **포인트 적립** | • 결제 전 휴대폰 번호 입력을 통해 스탬프를 적립하거나, 기존 스탬프 쿠폰을 적용하여 할인할 수 있다. | 중 |
| **FR-05** | **결제 처리** | • 신용카드 삽입형 결제 및 간편결제(삼성페이, 애플페이, 카카오페이 바코드)를 지원해야 한다. | 최상 |
| **FR-06** | **출력 기능** | • 결제 완료 시 영수증과 주문 번호가 인쇄된 대기표를 물리 프린터로 출력한다. | 상 |
| **FR-07** | **관리자 모드** | • 특정 바코드 인식 또는 비밀번호 입력을 통해 메뉴 관리, 품절 설정, 영수증 강제 출력, 당일 매출 조회가 가능한 관리자 화면으로 진입한다. | 상 |

---

## 4. 비기능적 요구사항 (Non-functional Requirements)


| 요구사항 ID | 구분 | 요구사항 상세 내용 | 중요도 |
| :--- | :--- | :--- | :--- |
| **NFR-01** | **사용성 (UI/UX)** | • 사용자의 시선 흐름을 고려하여 결제 버튼은 우측 하단에 크게 배치한다.<br>• 시니어 고객을 위한 '큰 글씨 모드' 전환 버튼을 메인 화면에 상시 노출한다. | 상 |
| **NFR-02** | **성능 및 반응성** | • 화면 터치 시 다음 화면 전환이나 옵션 선택 반영은 0.5초 이내에 완료되어야 한다.<br>• 60초 동안 화면 입력이 없을 경우 "대기 화면으로 돌아갑니다" 안내 팝업 후 자동 초기화된다. | 중 |
| **NFR-03** | **안정성 (예외처리)**| • 결제 중 카드 오류나 한도 초과 발생 시 정확한 실패 사유를 화면에 팝업으로 안내하고 장바구니 상태를 유지해야 한다.<br>• 프린터 용지(영수증)가 없거나 잼(Jam)이 발생하면 화면에 '점검 중(결제 불가)' 안내를 띄우고 점주에게 푸시 알림을 보낸다. | 최상 |
| **NFR-04** | **하드웨어 호환** | • 24인치 세로형 터치스크린 디스플레이(1080x1920 해상도) 환경에 최적화되어야 한다. | 상 |

---

## 5. 제약사항 및 외부 인터페이스
* **HW 연동:** IC 카드 리더기, 바코드/QR 스캐너, 영수증 열전사 프린터와의 드라이버 연동이 필요함.
* **통신 제약:** 매장 내 Wi-Fi 또는 LAN 선 연결이 끊겼을 때를 대비해 결제 시도 시 오프라인 에러를 명확히 처리해야 함.
